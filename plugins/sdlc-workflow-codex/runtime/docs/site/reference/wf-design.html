Design (/wf design)

Design is one /wf sub-command — /wf design — that runs as a compressed workflow. It is the design producer: it authors the design brief and the visual contract, runs the 15 visual transformations, and (for build commands) drives the downstream lifecycle stages itself. The standalone /wf-design router has been retired; everything it did is now an argument to /wf design.

The 21 design commands (sub-command: a keyword that selects a specific design operation) are arguments to this one key — never their own keys. The lifecycle stages slice / plan / implement / verify / review are the design consumers: each pulls the slice of design knowledge it needs from one shared library, gated behind a UI stack (stack.ui ≠ ∅).

Two invocation shapes

The first token after design is an optional slug, resolved by an exact existence check (never fuzzy): if .ai/workflows/<token>/00-index.md exists it is the slug; otherwise the token is the design command. The shape decides how far the flow travels.

ShapeFormEffect
In-workflow (slug given)/wf design <slug> <cmd> [instr]Compressed in-workflow flow. Produces the brief + contract, then drives slice → plan → implement → verify itself, compressed. Design owns the downstream flow — there is no hand-back to /wf slice.
No slug/wf design <cmd> [instr]Full lifecycle. Creates a new slug and runs the complete intake → … → retro flow, with the design intent seeding it.

Because the in-workflow shape continues straight into slice/plan/implement/verify rather than writing a brief and stopping, the design-consuming stages always fire — closing the old craft → implement skip.

Per-category flow span

The <design-command> resolves to a category, and the category decides which stages run. slice → plan → implement → verify is the span for build commands only.

CategoryCommandsIn-workflow span (slug given)No slug
Producercraft02b-design → 02c-craft → slice → plan → implement → verifycreate slug → full lifecycle
Transformationthe 15 (colorize, harden, typeset, adapt, …)focused 02c-craft → slice → plan → implement → verify (the transform is the implement step)create slug → full lifecycle
Review / analysisaudit, critiquesingle review step → 07-design-audit.md / 07-design-critique.md; no slice/plan/implementstandalone one-shot report
Inspectionextractsingle read-only step → design-notes/extract-<ts>.md + tokens-extracted.cssstandalone extract report
Contextsetup, teachwrite PRODUCT.md / DESIGN.md (repo root, slug-independent)identical — repo-root files, no stages

Only Producer and Transformation produce code, so they are the only categories that run the build span. The old "you cannot transform code that doesn't exist yet" gate inverts: the compressed flow now builds the code at its implement step, so a transform either creates the surface or modifies the slug's existing implementation.

Review & inspection

CommandWritesWhat
audit07-design-audit.mdTechnical-quality judgment (theming, responsive, anti-patterns) + 0–4 scoring. Consumes the a11y/perf/web-vitals already measured by verify in 06-verify-*.md rather than re-running axe-core. Also runs as a /wf review dimension.

Transformation (15 commands — applied at the implement step)

Each transformation modifies code, registers an add-on (add-on: an augmentation entry recording what changed and why) in 00-index.md, and writes design-notes/<sub>-<timestamp>.md.

CommandFor
animatePurposeful motion, micro-interactions, transitions, hover effects.
bolderAmplify a too-safe design — more visual character without breaking usability.
clarifyBetter UX copy, error messages, labels, instructions.
colorizeStrategic color where the surface is too monochromatic.
delightMoments of joy, personality, unexpected touches.
distillStrip to essence; remove unnecessary complexity.
hardenAccessibility — keyboard nav, screen-reader, contrast, focus management.
layoutComposition, spacing, visual rhythm; fixes monotonous grids.
onboardFirst-run UX — intro, empty states, tooltips, progressive disclosure.
optimizeUI performance — loading, rendering, animation cost, bundle size.
overdrivePush past convention with shaders, spring physics, 60fps animations.
polishFinal pre-ship pass — alignment, spacing, micro-detail consistency.
quieterTone down overstimulating designs while preserving quality.
typesetTypography — font, hierarchy, sizing, weight, readability.
adaptResponsive — breakpoints, fluid layouts, touch targets, cross-device.

The consumption gradient — how the stages read design

/wf design produces; these stages consume, each pulling only its slice of the shared design library (gated behind stack.ui ≠ ∅, so non-UI work is unaffected):

StageRelationshipWhat it pulls
slicestructures around itthe brief's state inventory + the contract's mock-fidelity inventory → candidate slice boundaries; loads onboard/polish state-completeness knowledge.
plancites itturns each recommended reference into a concrete plan-step pointer; carries register + anti-goals forward.
implementapplies itreads the transform playbook for each cited reference, applies it during the build, and registers a design-<sub> add-on. The deepest consumer.
verifymeasures ita11y / perf / responsive / web-vitals are the measurable design floor; per-augmentation re-checks confirm each transform hit its goal. Measured once here.
reviewjudges itdesign-audit + design-critique run as two dimensions in the parallel fan-out; audit interprets verify's numbers (single source of truth).

Technical details — augmentation registration

Every transformation command (plus audit and critique) registers itself in 00-index.md's augmentations: array. Each entry is type + artifact + created-at, with files-modified on transformations:

augmentations:
  - type: design-harden
    artifact: design-notes/harden-20260510T1432Z.md
    created-at: "2026-05-10T14:32:00Z"
    files-modified: [src/components/Button.tsx, src/styles/focus.css]
  - type: design-animate
    artifact: design-notes/animate-20260510T1530Z.md
    created-at: "2026-05-10T15:30:00Z"
    files-modified: [src/components/Card.tsx]

Handoff translates each registered entry into a reviewer-visible mention; ship translates each into a changelog line per the External Output Boundary rules.
