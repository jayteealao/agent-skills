Navigate existing workflows

You need to find a workflow you started, pick it back up after a break, see what to run next, or repair one that got stuck. This page shows the /wf keys that handle each of those situations.

Pre-conditionsAt least one workflow directory exists at .ai/workflows/<slug>/. A slug (a short hyphenated ID like auth-refresh) is the unique name the plugin assigned when you created the workflow.
ToolAll commands on this page are keys of /wf — the single entry point. Navigation and maintenance of existing workflows live here (the former standalone /wf-meta router is retired).

Find: see everything at a glance

You opened the repo and you are not sure what is in flight.

/wf status

Lists every active workflow, the current stage (one of ten named steps a workflow moves through), the recommended next command, and any open questions. Use this as your daily entry point before touching anything else.

To zoom in on one workflow, pass its slug:

/wf status auth-refresh

The single-workflow view shows the full 10-stage progress map with a status for each stage (not started / in progress / complete / skipped), the slice statuses, and recent activity. Workflows with open questions appear in the Blocked dashboard group. A slice is one independently deliverable chunk of the workflow — a named unit of work inside the larger feature.

Resume: pick up where you left off

You walked away from a workflow hours, days, or weeks ago and you need to get back into it.

/wf recap auth-refresh

Reads the workflow's full artifact trail — every note, plan, and decision the plugin has written under .ai/workflows/auth-refresh/ — identifies the next unfinished step, and proposes a concrete command you can run immediately. Use this any time context-switching has left you uncertain where things stand. (recap is the renamed former resume.)

Next: get the next command without a full re-read

You know the workflow well and you just want to know what to run now.

/wf status auth-refresh

Lighter than recap. The single-workflow status view ends with a ## Next block naming the immediate next command, without re-reading the complete artifact trail. Good when you have been working in the workflow recently and just need a quick reminder of the immediate next action. (This absorbs the former /wf-meta next.)

Repair: fix a stuck or drifted workflow

Something feels off — maybe the 00-index.md control file disagrees with the actual files on disk, or you pulled a branch that added new artifacts and the index was not updated.

/wf status auth-refresh

The status detail view reconciles the 00-index.md control file against the artifacts on disk automatically. For a deeper reality-drift report, add deep: /wf status auth-refresh deep. Run this after manual edits, after pulling a branch that landed new artifacts, or whenever status output looks wrong. (This absorbs the former /wf-meta sync.)

The /wf navigation & lifecycle keys
What you wantCommand
"What am I working on?"/wf status
"Where did I leave off in auth-refresh?"/wf recap auth-refresh
"What's the next command for auth-refresh?"/wf status auth-refresh
"Index file looks wrong — repair it."/wf status auth-refresh (deep report: /wf status auth-refresh deep)
"Explain a workflow artifact or review findings."/wf recap auth-refresh plan
"Ask a code question, explore the codebase, or commission deep web research."the deep-research skill
"Generate a Diátaxis announcement from a shipped workflow."/wf ship auth-refresh announce
"Correct a wrong spec or add new scope."/wf intake auth-refresh <scope> (there is no amend)
"Terminate one slice so downstream is satisfied."/wf close auth-refresh <slice>
"Archive a finished or abandoned workflow."/wf close auth-refresh <reason>
"Author the project ship plan (once)."/wf ship-plan init
"Build the CI/CD + DX pipeline from the plan."/wf ship-plan build
