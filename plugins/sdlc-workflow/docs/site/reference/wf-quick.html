Quick & standalone flows

The standalone /wf-quick router has been retired. Its ten flows are now distributed across /wf: eight become intake modes — arguments to /wf intake <mode> — and two (probe and simplify) become their own top-level /wf keys. A ninth intake mode, adopt, was added later, so /wf intake <mode> now takes nine modes in all. You reach for these when you have a small, well-defined change that does not need the full 10-stage lifecycle.

The new shape

The first token after intake is an optional mode, resolved by an exact keyword check: if the token matches one of the nine mode names it selects that compressed flow; otherwise the token is the description of the work. The mode decides how compressed the intake run is.

ShapeFormEffect
Standard intake/wf intake "<description>"Full intake. Runs the standard 10-stage lifecycle starting at Stage 1.
Mode intake/wf intake <mode> "<description>"Compressed intake. Change-mode lifecycles (fix, hotfix, refactor, update-deps) run all stages in a single-pass, lightweight flow — no stage is skipped. Terminal-analysis modes (rca, investigate, discover, ideate) produce a single finding artifact without starting a build pipeline.
Slug-mode intake/wf intake <slug> <mode>Compressed in-workflow flow. Attaches a compressed slice to an existing slug rather than creating a new workflow.

The nine intake modes

Each mode maps to a compressed flow that collapses or skips stages relative to the full pipeline.

ModeYour situationWhat it compresses
fixOne file. One change. The fix is obvious.Shape + plan merged into a single 01-fix.md; routes to /wf implement.
hotfixProduction is broken right now; speed beats review.Full standard lifecycle expedited under incident pressure: scoped to minimum viable fix, production-branch base. Review defaults to security rubric (not bypassed). Writes 01-hotfix.md, 02-shape.md, 03-slice.md, 04-plan.md then routes to /wf implement.
rcaYou have a bug report but don't know the cause yet (static: read code & history).Single 01-rca.md artifact with parallel diagnosis sub-agents. Recommends downstream command.
investigateYou know there's a problem but want 2–3 distinct approaches before committing.Single 01-investigate.md with A/B/C options + tradeoff table. No winner picked.
discoverYou have a theory about how the code works and want it verified or falsified.Single 01-discover.md with FOR/AGAINST/counter-hypothesis evidence and verdict.
update-depsDependency audit — security patches, major-version bumps, safe minor updates.Tiered scan → research → plan → implement → verify under .ai/dep-updates/.
refactorBehaviour-preserving restructure with a test-suite baseline captured first.Full standard lifecycle with behavior-preserving discipline. Captures test baseline in 02-shape.md before touching code, plans incremental green steps. Writes 01-refactor.md, 02-shape.md, 03-slice.md, 04-plan.md; routes to /wf implement. Review defaults to refactor-safety.
ideateBrainstorm improvement candidates for a focus area before sprint planning.Single artifact under .ai/ideation/ with ranked candidates and effort estimates.
adoptWork already done outside the lifecycle that you want tracked retroactively.Reconstructs the workflow trail for existing changes and marks the workflow provenance: adopted, so already-shipped or in-progress work joins the lifecycle without re-doing it.

Slug-mode — compressed-slice attach

When the first argument after intake is an existing slug (exact existence check: .ai/workflows/<token>/00-index.md exists), the mode runs against that slug instead of creating a new workflow. It writes a single compressed slice into the slug's directory and leaves the workflow's stage, branch, and progress untouched.

/wf intake my-feature-abc1 fix

The slug-mode argument order is always slug first, mode second. The mode word after the slug is the compressed flow; anything ambiguous (a token that could be either a description or a mode name) is resolved by the existence check first.

Suggest-and-confirm auto-routing

When you run /wf intake "<description>" without a mode, intake reads the description and the context and may suggest a mode: "This looks like a fix — run as /wf intake fix?" You confirm or decline. Auto-routing never fires silently; it always asks.

/wf probe — runtime-truth verification v9.14

/wf probe is a top-level /wf key, not an intake mode. It is slug-mode only — it refuses to run without an existing slug. Drives the running artifact, captures observable output (screenshots, stdout, response bodies, log lines), and writes findings without writing a fix.

FormEffect
/wf probe <slug>Slug-wide sweep — probe every AC across every slice in the slug.
/wf probe <slug> <target>Focused probe on the target string. There are no flags — probe takes a slug and an optional target string.

See the /wf reference for the full probe argument grammar, four-layer target resolution, deferral-clearing behaviour, and multi-adapter details.

/wf simplify — review-and-route triage

/wf simplify is also a top-level /wf key. It aggregates findings across three rubrics (Code Reuse, Code Quality, Efficiency), asks you which to accept, and classifies each accepted finding by routing it to the appropriate downstream command. It never writes code.

ScopeFormWhen to use
branch (default)/wf simplifyPre-PR sanity pass on everything new on this branch.
commit/wf simplify commit <sha>Triage a specific commit before pushing.
plan/wf simplify plan <slug> <slice>Catch over-engineering before /wf implement.
codebase/wf simplify codebase [<path>]Periodic codebase audit.
