Hooks reference

The plugin wires five Node hooks across three events (one SessionStart, one PreToolUse, three PostToolUse) via hooks/hooks.json. Each runs from the committed esbuild bundle under dist/ — not the hooks/*.mjs source — so a fresh install runs dependency-free with no npm install. (Earlier releases shipped shell + Python equivalents under hooks/scripts/; those were retired in v9.34.3–v9.34.5, and the no-op PreCompact hook was removed in v9.41.0. Validation, verification, staging, rendering, and orientation are now entirely native Node.)

pre-write-validate (PreToolUse: Write)

Fires before a Write to */.ai/workflows/<slug>/*.md. Blocks writes that violate:

  • Filename convention — NN-stagename.md / NNa-stagename.md (substages like 02b-design.md), plus the allowlist risk-register.md, estimate.md, announce.md, po-answers.md, and anything under design-notes/.
  • YAML frontmatter — must exist between --- markers with schema, type, slug (the po-answers.md prose log is exempt).
  • Schema version — schema: sdlc/v1.
  • Slug stability — frontmatter slug must match the directory name.

Note: .ai/workflows/INDEX.md (the registry) and .ai/ship-plan.md (project-level) are handled separately — the per-slug path pattern doesn't match them.

post-write-verify (PostToolUse: Write/Edit/MultiEdit/NotebookEdit)

Fires after the write, reading the file from disk. Runs deep Ajv validation of the frontmatter type + required fields against tests/frontmatter.schema.json for artifacts under .ai/workflows/, .ai/simplify/, .ai/profiles/, and .ai/docs/. The po-answers.md prose log is exempt. Exits 2 with diagnostics on failure so the model can repair the frontmatter in place.

Sibling-fragment hard block v9.47

For rich-tier artifact types, post-write-verify exits 2 (blocks the write) when the .md lands without its mandatory sibling .yaml. The 13 gated types are review, plan, design, ship-run, rca, benchmark, experiment, instrument, profile, simplify-run, review-command, design-audit, and design-critique (the last three added v9.48). Two escape hatches bypass it: set fragment: none (also accepts skip / n/a) in the artifact's frontmatter to exempt a single file, or set hooks.remindMissingFragments: false in .ai/sdlc-config.json to disable it project-wide. The automation-regenerable types sync-report and docs-index are intentionally excluded so a hard block can't wedge their regenerators.

post-write-auto-stage (PostToolUse)

Auto-stages changed code with git add when an active workflow is in the implement stage and the branch strategy is dedicated or shared. Honors .ai/.no-auto-stage.

post-write-render (PostToolUse)

Best-effort: re-renders the affected part of the sunflower HTML view under .ai/_view/ after an artifact changes. Never blocks the write.

session-start-orient (SessionStart)

Fires once per session and re-fires after a context compaction (source: compact). Reads .ai/workflows/*/00-index.md, surfaces active workflows as session context, and kicks off detached bootstrap rendering of the view. Because it re-reads workflow state from disk after compaction, it is also what re-orients Claude once the conversation is compressed.

Configuration — the hooks block

Per-repo, in .ai/sdlc-config.json. All default to true:

KeyDefaultControls
autoStagetruepost-write-auto-stage git add behaviour.
validateOnWritetruepre-write-validate filename / frontmatter checks.
verifyOnWritetruepost-write-verify deep Ajv validation.
remindMissingFragmentstrueThe sibling-fragment hard block + reminder. false disables both.

Why dist/ — the build model

Hooks run from dist/*.mjs: committed esbuild bundles with their third-party dependencies inlined. That is what lets a fresh install run with no runtime npm install. The source lives in hooks/; after editing any hook (or anything under lib/, renderers/, components/) you must run npm run build to refresh dist/, or a CI freshness gate fails and production runs stale code. See Build & dist model.
