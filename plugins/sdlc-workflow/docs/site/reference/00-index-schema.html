00-index.md schema

The control file. Pure YAML frontmatter, no markdown body. Every stage reads it; every stage that completes updates it.

Required fields

FieldTypeMeaning
schemaliteralsdlc/v1
typeliteralindex
slugstringStable identifier; matches directory name.
titlestringHuman-readable task name.
statusenumactive / complete / blocked / abandoned
current-stagestringMost recently started stage name.
stage-numbernumber1–10.
updated-atISO 8601Real timestamp from date -u.
selected-slice-or-focusstringCurrently active slice slug.
open-questionsarrayUnanswered questions blocking progress.
next-commandstringCommand name (e.g., wf-shape).
next-invocationstringFull slash command, ready to copy.
workflow-filesarrayAll artifacts written so far.
progressmap10 stage names → not-started / in-progress / complete / skipped / blocked
slicesarrayPer-slice summary objects (slug, status, complexity, depends-on).
branch-strategyenumdedicated / shared / none
branchstringFeature branch name (default feat/<slug>).
base-branchstringBranch to merge back into.

Optional fields (set as stages run)

pr-urlSet by handoff.
pr-numberSet by handoff.
augmentationsArray of { kind, slice, ref, registered-at } objects.
workflow-typeFor compressed flows: quick, rca, investigate, hotfix, refactor, update-deps, discover, ideate.
compressed-slices(v9.9.0; trigger reshaped in v9.10.0) Array of { slug, slice-type, created-at } objects, one per compressed slice landed on this workflow via /wf-quick <sub> <slug> ... (positional slug; the v9.9.0 --slug flag was removed in v9.10.0). Created lazily on first slug-mode write; absent on workflows that never received one. slice-type is the originating sub-command (quick, rca, investigate, discover, hotfix, update-deps, refactor, ideate, simplify). The slice file itself lives at 03-slice-<slice-slug>.md and is also listed in workflow-files; the compressed-slices array is the explicit registry that distinguishes compressed slices from formally-planned ones. See wf-quick slug-mode.

Sibling registry file (v9.10.0)

.ai/workflows/INDEX.md sits at the workflow root (sibling of every slug directory). It is not a workflow artifact and has no YAML frontmatter — it is a registry/cache. Format: one header comment line, then one tab-separated row per workflow with columns slugstatusworkflow-typebranchupdated-at, sorted alphabetically by slug. Closed workflows are retained with status column = closed for history. The registry is the lookup substrate for /wf-quick's positional slug detection (the dispatcher runs grep -P "^<token>\t" INDEX.md to decide between slug-mode and standalone). Maintained by /wf-meta sync (bootstrap on first run; refresh on every subsequent run) and additively touched (updated-at column only) by every /wf-quick slug-mode write. See skills/wf-meta/reference/sync.md Step -1 for the format and procedure.
