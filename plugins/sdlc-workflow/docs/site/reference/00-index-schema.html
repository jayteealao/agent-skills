00-index.md schema

The control file. Pure YAML frontmatter, no markdown body. Every stage reads it; every stage that completes updates it.

Required fields

FieldTypeMeaning
schemaliteralsdlc/v1
typeliteralindex
slugstringStable identifier; matches directory name.
titlestringHuman-readable task name.
statusenumactive / complete / blocked / abandoned
current-stagestringMost recently started stage name.
stage-numbernumber1–10.
updated-atISO 8601Real timestamp from date -u.
selected-slice-or-focusstringCurrently active slice slug.
open-questionsarrayUnanswered questions blocking progress.
next-commandstringCommand name (e.g., wf-shape).

Optional fields (set as stages run)

pr-urlSet by handoff.
pr-numberSet by handoff.
augmentationsArray of { kind, slice, ref, registered-at } objects.
workflow-typeFor compressed flows: quick, rca, investigate, hotfix, refactor, update-deps, discover, ideate, simplify.
compressed-slicesArray of { slug, slice-type, created-at } objects. Appended by /wf-quick sub-commands when run in slug-mode. slice-type values: quick, rca, probe v9.14, investigate, discover, hotfix, update-deps, refactor, ideate, simplify.
runtime-evidence-deferrals v9.14Array of { slice, reason, deferred-at, cleared-by } objects. Appended by /wf verify when a slice's user-observable AC is annotated with interactive-verification: deferred. /wf ship Step 6.5 hard-blocks while any entry has cleared-by: null. Cleared by a /wf-quick probe run whose captured evidence satisfies the deferred AC (sets cleared-by: probe-<descriptor>) or by a verify re-run in a capable environment.

v9.14 runtime-evidence-deferrals shape

runtime-evidence-deferrals:
  - slice:              # the slice whose verify deferred a user-observable AC
    reason:   # mirrors interactive-verification-defer-reason from the verify artifact
    deferred-at:           # when verify wrote the deferral
    cleared-by: null                 # null until cleared; probe- when probe satisfies it

An entry never moves out of the array — clearing is recorded by mutating cleared-by from null to the clearing artifact's identifier. This preserves the audit trail: a future reader can see which slices once had open deferrals and how each was eventually closed. The /wf-meta status Runtime column derives its display from this field (count of cleared-by: null entries → deferrals: N).
