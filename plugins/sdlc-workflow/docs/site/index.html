sdlc-workflow

A Claude Code plugin that turns every change — feature, fix, refactor, spike — into a reproducible sequence of stages, each writing a durable, machine-readable artifact next to your code.

Why this exists AI conversations don't remember. The next session starts blank. Every decision the AI made — about scope, architecture, edge cases, test strategy — vanishes with the context window. sdlc-workflow makes the AI's reasoning visible and persistent: every stage reads what the previous wrote, asks you to fill in the gaps, and commits its output to a YAML-fronted markdown file under .ai/workflows/<slug>/. Three weeks later, you can still see why a particular approach was chosen.

  slice --> plan[4·plan]

  impl --> verify[6·verify]
  verify --> review[7·review]
  review --> handoff[8·handoff]
  handoff --> ship[9·ship]
  ship --> retro[10·retro]

  classDef def fill:#e8f0fe,stroke:#1d4ed8,stroke-width:1px,color:#1f1f1d
  classDef opt fill:#f5e8fe,stroke:#7c3aed,stroke-dasharray:3 3,color:#1f1f1d
  class intake,shape,slice,plan,impl,verify,review,handoff,ship,retro def
  class design opt

Every stage reads what previous stages wrote and writes its own artifact. Adaptive routing means you can also skip forward, loop back, or branch into parallel slices — the diagram above is the spine, not a railroad.

The four-quadrant docs (Diátaxis)

This site follows the same documentation framework the plugin itself preaches. Each page declares its quadrant in a colored badge at the top, and the body follows that quadrant's conventions:

QuadrantYou're here because…The pages look like…
tutorialYou want to learn by doing. You don't yet know what you don't know.Imperative steps. Stated outcomes. One worked example end-to-end.
how-toYou know what you want to accomplish. You need a recipe.Pre-conditions + numbered steps + variations.
referenceYou need to look something up.Information-oriented. Alphabetised. Exhaustive. No opinions.
explanationYou want to understand why.Discursive prose. Trade-offs. History. Comparison with alternatives.
tipsYou're past basics. You want sharp edges and shortcuts.Short opinionated entries. Anti-patterns. FAQ.

If a page mixes shapes — "let me explain a concept while you copy these steps" — the framework says it has failed both audiences. The site holds to that discipline.

What's new

v9.6.0

  • /wf-quick simplify entrypoint — three-agent review-and-route triage (Code Reuse, Code Quality, Efficiency) across four scopes: branch (default), commit <sha>, plan <slug> <slice>, codebase [<path>]. Never writes code — classifies each finding and routes it to the appropriate downstream command (/wf-quick fix, /wf-quick refactor, /wf intake, /wf-meta amend, etc.). Adapted from the Claude Code bundled simplify skill but realigned to the plugin's orchestrator discipline. See the simplify section.

v9.5.0

  • Handoff PR-readiness block — commitlint, public-surface drift, doc-mirror regen, a bounded PR-comment triage loop, rebase onto base, and a live PR-state check. Produces a single readiness-verdict that ship gates on. See Triage PR comments and The readiness gate.
  • Ship plan + per-release runs — a project-level .ai/ship-plan.md (authored once via /wf-meta init-ship-plan) drives a 13-step replayable ship sequence. Runs accumulate as 09-ship-run-<run-id>.md. Partial failures resume rather than restart. See Author a ship plan and Run a release.

Local preview & layout

The site is plain static HTML. Open index.html directly, or serve it locally:

cd plugins/sdlc-workflow/docs/site
python3 -m http.server 8000
# open http://localhost:8000

See the source tree for the directory layout. Diagrams use Mermaid via CDN; when the CDN is unreachable, each diagram's source text remains readable as a fallback.
