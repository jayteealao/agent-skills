Contributing causes

Process · Load-test gate not enforced
Slices touching PaymentForm.tsx should require a load-test pass before merge. The rule existed in the CI config but was opted-out for "trivial UI changes"; rev 2 was misclassified.
Infra · Pool sized for steady state, not hydration storms
The backend WebSocket pool is 250. Hydration of a popular page can briefly open more than 250 sockets even under good code; under bad code, recovery is impossible.
Observability · Alert fires after user impact, not before
Confirm-latency alert is at 10s, but the client timeout is 8s. The alert can only fire after the client has already given up. Threshold should be 6s.

Risks

High · Region detection is silently wrong
useRegion() reads from a cookie set by the CDN at the edge. The cookie lags geo-IP updates by ~5min, and travelling users carry it across borders. A user in France can briefly see USD pricing on first session. Mitigation: surface a region badge in the checkout summary so the mismatch is visible to the user.
Medium · Tax surcharge calc moves server-side
The old client-side LegacyCurrency.ts did naive surcharge math. regionPolicy.ts moves this to the server, which means the cart summary may briefly show a stale total on first render. Mitigation: optimistic placeholder + reconcile in the useCart effect.
Low · Bundle size +3.2 KB gz
The new region YAML + lookup adds ~3 KB gz to the client bundle. Within budget; flagged so the perf dashboard expects it.
